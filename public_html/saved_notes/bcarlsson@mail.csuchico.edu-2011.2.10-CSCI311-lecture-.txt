Replacement Selection<br><br><ul><li dir="ltr">for <i>external</i> sorting (sorting elements in secondary storage)<br></li><li dir="ltr">recall:<br><ul><li dir="ltr">given <span style="font-family:monospace">N</span> units of storage (secondary) and <span style="font-family:monospace">m</span> units of RAM, we load runs (chunks) of size <span style="font-family:monospace">m </span>into memory and sort them<br></li><li dir="ltr">we end up with a total of <span style="font-family:monospace">N/m</span> runs each of size <span style="font-family:monospace">m</span></li><li dir="ltr">in replacement selection, we treat RAM as a priority queue<br></li></ul></li><li dir="ltr">replacement selection keeps <span style="font-family:monospace">m</span> units worth of nodes in RAM in a min-heap, and when RAM is full, we <span style="font-family:monospace">pop</span> the heap into the new (sorted) secondary storage area, and insert the next node into the heap (into RAM) from secondary storage.<br></li><li dir="ltr">the result is a constantly-changing heap, cycling nodes consistently</li></ul>
